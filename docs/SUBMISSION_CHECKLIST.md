# Abstract Submission Quick Checklist

Use this checklist to verify your abstract submission is complete and compliant before submitting.

---

## Pre-Submission Checklist

### Personal Information (ABSTRACT_SUBMISSION.md)
- [ ] Student Name filled in
- [ ] Roll Number filled in
- [ ] Program specified (B.Tech/M.Tech/etc.)
- [ ] Department specified
- [ ] Email address provided

### Supervisor Information (ABSTRACT_SUBMISSION.md)
- [ ] Main supervisor name filled (MUST be from your institute)
- [ ] Supervisor designation specified
- [ ] Supervisor department specified
- [ ] Supervisor email provided
- [ ] External co-guide information (if applicable)

### Content Requirements
- [ ] Project title is clear and descriptive
- [ ] Abstract section is complete (all 10 subsections)
- [ ] Problem statement clearly articulated
- [ ] Research objectives listed (at least 3-5)
- [ ] Methodology described in detail
- [ ] Technical specifications provided
- [ ] Expected outcomes quantified
- [ ] Timeline included (6+ months)
- [ ] References properly formatted (at least 5-8)

### Compliance Requirements

#### Research-Based Work
- [ ] Project demonstrates original research component
- [ ] Literature review conducted and documented
- [ ] Novel contributions or implementations identified
- [ ] Not just software development - includes analysis
- [ ] Research methodology clearly described

#### Project Type
- [ ] Clearly stated as Major Project (not internship)
- [ ] Substantial duration (6+ months)
- [ ] Comprehensive scope demonstrated
- [ ] Independent work with supervision

#### Supervision Structure
- [ ] Main supervisor is from your institute (verified)
- [ ] Supervisor has agreed to supervise
- [ ] External co-guide role clearly defined (if any)
- [ ] Supervision structure is compliant

#### Plagiarism Compliance (<10%)
- [ ] All code is original implementation
- [ ] External libraries properly cited
- [ ] No copied code without attribution
- [ ] Plagiarism check completed
- [ ] Report shows less than 10%
- [ ] Report saved as evidence

#### AI-Generated Content (0% Tolerance)
- [ ] No AI-generated code used
- [ ] All code written manually
- [ ] Can explain every line of code
- [ ] Can demonstrate understanding
- [ ] Ready to defend implementation choices
- [ ] Development history available (Git commits)

#### Institute Guidelines
- [ ] Follows prescribed format
- [ ] Meets word count requirements
- [ ] Uses approved template (if provided)
- [ ] All sections complete
- [ ] Professional formatting

### Declaration and Signatures
- [ ] Declaration section read and understood
- [ ] All 8 declaration points apply to your work
- [ ] Student signature provided
- [ ] Date filled in
- [ ] Supervisor approval obtained
- [ ] Supervisor signature received
- [ ] Institute stamp (if required)

### Supporting Documents
- [ ] Plagiarism report (<10%) attached
- [ ] Supervisor approval letter obtained
- [ ] Timeline/Gantt chart prepared (optional)
- [ ] Initial research notes available

### Document Quality
- [ ] No spelling errors
- [ ] No grammatical errors
- [ ] Consistent formatting throughout
- [ ] Professional appearance
- [ ] All links and references working
- [ ] PDF version created (for submission)

### Submission Requirements
- [ ] File named correctly (follow institute guidelines)
- [ ] Submitted to correct portal/email
- [ ] Submitted before deadline
- [ ] Confirmation email received
- [ ] Backup copy saved
